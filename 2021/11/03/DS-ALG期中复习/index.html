P1 循环不变式

  • 要想证明BubbleSort确实对原先的元素进行了排序，我们就还要证明经过排序的数组中的元素都是原数组中的元素。

  • 循环不变式性质：

    初始化：循环的第一次迭代前，它为真。

    保持：如果循环的每一次迭代前它为真，那么下次迭代前它仍为真。

    终止：在循环终止时，不变式为我们提供一个有用的性质，该性质有助于证明算法是正确的。

  • 如何寻找循环不变式：

    算法是一步一步执行的，如果每一步（包括初始和结束）都满足同一个条件，那么这个条件就是要找的循环不变式。

    每一层循环都有不同的循环不变式，要注意每一次循环改变了什么，达成了什么目的。

  • 循环不变式的证明：

    （basis）在第一次迭代前，不变式成立

    （inductive step）假设在某一次循环前成立的，那么在本次循环之后，性质仍然保持。

    （termination）程序可以终止而且可以得到想要的结果。

P3 从中缀表达式到后缀表达式

  • 主要是利用一个栈实现

  • 如果输入是一个操作数则直接输出，如果是一个操作符，如果栈此时是空的则将符号压入栈中，如果栈非空，则将符号的优先级和栈中的符号的优先级比较，将所有优先级更高或者优先级相同的符号pop出来输出，然后将操作符压入栈中。

  • 具体实现：输入一个string infix，输出一个vector postfix，用一个栈保存操作符。

  • Infix2Postfix(infix)
	vector postfix;
	Stack S;
	for i from 0 to infix.length()-1 do
		if '0' <= infix[i] and infix[i] <= '9' then
			postfix.push_back(infix);
		else
			while !S.empty() and S.top() < infix[i] do
				postfix.push_back(S.pop());
			end while
			S.push(infix[i]);
		end if
	end for
	while !S.empty() do
		postfix.push_back(S.pop());
	end while
	return postfix;

  • 插入排序时间复杂度和数组中逆序对个数的关系

    假设逆序对有m个，时间复杂度就是$\Theta(n+m)$

    证明过程看一下：

    设$f_i=|{j:j<i$ $and$ $A_j>A_i}|$

    显然：$m=\sum_{i=1}^{n}f_i$

    根据插入排序的循环不变式可以知道，每一次循环过后A[1…j]都是已经排好序的，所以假设这些元素里存在逆序对且由于插入排序已经回到了正确的位置，那么这些元素一定就是A[j-f(j)…j-1]，也就是现在在key的右边的元素，一定是和key组成逆序对的，所以循环条件判断执行了f(j)+1次，而调换元素顺序一定执行了f(j)次。总共的时间复杂度应该就是$\sum_{i=1}^n(c_2f(j_0)+1)+c_3f(j_0)+c_4f(j_0))=\Theta(n+m)$

  • 统计逆序对的算法

    Merge的过程：将数组分成左右两个部分，开两个新的工具数组存放左右两边的数组，模仿mergesort进行扫描，只是当左边有数字小于右边，也就是要将右边元素放进原A数组的时候，要用cnt对逆序对进行计数，返回扫描过程中的逆序对数，同时将两个数组合并为有序数组。

P6 最大子数组的分治算法

根据提示，应该修改添加一些返回信息来简化combine的过程。

  • 复习书上的做法

    最大子数组有三种情况：完全位于原数组的左半边；完全位于原数组的右半边；跨越原数组的中点的子数组中和最大的。所以只要找出来进行比较即可。

    前两种我们都可以递归解决，因此额外要解决的就是第三种情况。

    第三种情况采取遍历的方法解决，先找出左边子数组以mid为结束点的最大子数组，再找出右边子数组以mid+1为起点的最大子数组然后合并即可。

    接下来是分治策略的核心，递归过程。先对左右两边进行递归，注意这里除了返回最大子数组的两个边界还应该返回子数组的和，因为有三种情况而比较的依据就是这个和。然后再用上面实现的算法找出第三种情况下的最大子数组，比较三个和得出答案。

  • 算法的修改，优化最后的combine也就是比较过程。sample里给出的答案是在最后一种情况找子数组的时候进行了修改。考虑可不可以在对左右子数组进行递归的时候顺便也把第三种情况也解决了？如果在递归的时候，函数也能返回左边子数组以mid为结束点的最大子数组和右边子数组以mid+1为起点的最大子数组，那就可以直接得到第三种情况要求的子数组。

    但是和书上的方式又有很大的不同。

    可以将最大子数组划分为三种情况：以最左边为起点的；以最右边为起点的；剩下的。

    每次函数的返回值为以上三种形式得到的最大子数组以及数组中元素的和。

    以最左边为起点的最大子数组的情况包含了教材方法中的子数组完全在左子数组中的还有以最左边为起点但跨越了中点的情况，以最右边为起点的最大子数组同理。如此便可以囊括了所有情况。

P7 遍历二叉树中的节点

要求在O(n)时间内遍历所有节点。

经典策略：Morris遍历

  1. 如果当前节点的左孩子为空，则输出当前节点并将其右孩子作为当前节点。

  2. 如果当前节点的左孩子不为空，在当前节点的左子树中找到当前节点在中序遍历下的前驱节点。

a) 如果前驱节点的右孩子为空，将它的右孩子设置为当前节点。当前节点更新为当前节点的左孩子。

b) 如果前驱节点的右孩子为当前节点，将它的右孩子重新设为空（恢复树的形状）。输出当前节点。当前节点更新为当前节点的右孩子。

  1. 重复以上1、2直到当前节点为空。

P2 二叉搜索树转换为hb二叉搜索树

复习一下树的旋转：

将一个节点上移、一个节点下移。树旋转会改变树的形状，因此常被用来将较小的子树下移、较大的子树上移，从而降低树的高度、提升许多树操作的效率。

  • 主要思路：

    先通过不断的右旋将BST转化成一条chain，再将得到的链不断左旋转化成一棵树。

  • 证明：

    定理：至多n-1次右旋可以将BST从一棵树转化成一条右走向链。

    Basis：n=1时不需要旋转

    I.H：假设$1\leq n\leq k$时，最多需要n-1次右旋

    I.S：当n=k+1，假设左边子树的大小是m，右边子树的大小是k-m

    ​ 根据I.H，最多需要$t=(m-1)+max{0,(k-m-1)}\leq k-1$次右旋

P3 将随机二叉树转化为BST

先用$O(n)$次旋转将二叉树转化成一条每个节点只有一个左子节点的链。

用一个冒泡排序的类似方法。

在链中可以通过一次swap，一次左旋和一次swap操作交换两个节点的位置但是不影响其他性质。

假设设置当前节点为X，其父节点Y（如果链躺下来就算是右边的）比它小的话。先将X和其兄弟节点（空）交换位置，再将Y左旋（降低高度），再将Y此时的右子树与左子树（空）交换位置；如果Y更大，则更改当前节点为Y。一轮下来，根节点应该就是所有节点中最大的一个节点。
