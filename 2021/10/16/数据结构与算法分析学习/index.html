Sorting

Characteristics of sorting algorithms

  • In-place:

    只需要$O(1)$的额外的空间。

  • Stability:

    相同大小的元素的相对位置没有改变。

Selection Sort

  • 思路

    1. 找出最小的元素。
    2. 对剩下的元素递归执行第一步。
    3. 当只剩一个元素时，直接返回该元素，递归终止。

  • Correctness

    循环不变式：After the ith iteration, the first i items are sorted, and they are the i smallest elements in the original array.

  • 时间复杂度

    $\Theta(n^2)$

  • 空间复杂度

    $O(1)$,thus in place.

  • Stability

    Not stable.Swap operation can mess up relative order.

Bubble Sort

  • 思路

    1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
    2. 对每一对相邻元素作同样的工作，从开始第一对到结尾的最后一对。这步做完后，最后的元素会是最大的数。
    3. 针对所有的元素重复以上的步骤，除了最后一个。
    4. 持续每次对越来越少的元素重复上面的步骤，直到没有任何一对数字需要比较。

  • Correctness

  • 时间复杂度

    $\Theta(n^2)$

  • 空间复杂度

    $O(1)$

  • Stability

    Stable

  • Improvement

    • 在迭代过程中可能在某一次迭代时就已经完成了排序，那么后面接下来的迭代可能就不再有必要，试图通过设置一个bool值来避免部分迭代的过程。

      在数组本就有序的最好情况下，只需要$O(n)$即可得到结果，避免了无用功。

      但是在最坏情况下时间复杂度仍然是$\Theta(n^2)$，即若数组是逆序的话，经历所有迭代避无可避。

    • 在n的值的改变上再贪心一点，因为每轮内层循环的时候，最后一次交换之后的元素一定是排好序的，所以可以将下一轮循环的范围缩小更多。

Quick Sort

  • 思路

    1. 挑选基准值：从数列中挑出一个元素，称为“基准”（pivot）。
    2. 分割：重新排序数列，所有比基准值小的元素摆放在基准前面，所有比基准值大的元素摆在基准后面（与基准值相等的数可以到任何一边）。在这个分割结束之后，对基准值的排序就已经完成。
    3. 递归排序子序列：递归地将小于基准值元素的子序列和大于基准值元素的子序列排序。递归到最底部的判断条件是数列的大小是零或一，此时该数列显然已经有序。
    4. 一般使用最后一个数字作为pivot。

    2. 扫描数组，将小于基准的放在左半边，大于基准的放在右半边。

    3. 最后把基准数放在最后留下的位置上。

Upper bound and Lower bond

  • **Upper bound: **

    • how fast can we solve the problem?

    • 针对算法本身，指对某种算法在最坏情况下需要的时间复杂度。

    • 每一种算法都有一个upper bound。

  • Lower bound:

    • how slow solving the problem has to be?

    • 针对问题本身，指针对该问题最快的算法在最坏的情况下需要的时间复杂度。

    • 任何算法都至少要花这么多时间去解决这个问题。

    • Larger lower bound is a stronger lower bound. (On the other hand, smaller upper bound is better.）

    • how do we prove a lower bound?!rely on structures/properties of the problem itself…

Bucket Sort

  • 思路：

    1. 创建“桶”。
    2. 扫描待排序的数组，插入到对应的“桶”的尾部。
    3. 按“桶”的标号按序输出“桶”内的数字。

  • 时间复杂度

    $\Theta(n)$

  • 不是一个基于比较的排序算法

  • 扩大“桶”的容量：在combine之前先对每个“桶”内的内容进行排序

    时间复杂度：$O(n+k+(n^2/k))$

  • stable

Radix Sort

  • 思路：

    1. 将所有数字的位数统一（高位补0）。
    2. 从最低位开始向高位依次进行排序。

  • Correctness（归纳法证明循环不变式）

    Claim: after ith iteration, items are sorted by their rightmost i bits.

    Use induction to prove the claim.

    [Basis] The claim holds after the first iteration.

    [Hypothesis] Assume the claim holds after the first k−1 iterations.

    [Inductive Step] Consider two items a and b after k iterations.

    • W.l.o.g., assume a appears before b. Thus, a[k]≤b[k].
    • If a[k]<b[k], then it must be a[k…1]<b[k…1].
    • If a[k]=b[k], since we use stable sort, it must be a[(k−1)…1]≤b[(k−1)…1]. Again, a[k…1]≤b[k…1].

    时间复杂度：$O(dn)$

    因为是十进制数，所以需要十个桶就足够：$\Theta(1)$

Trees

Introduction

a tree is either empty, or has a root connecting to the roots of zero or more non-empty subtrees.

连接的、非循环的无向图。

我们常使用的树是有根树。

  • 树的递归定义

    A tree is either empty, or has a root r that connects to the roots of zero or more non-empty (sub)trees.

    • Root of each subtree is a child of r, and r is the parent of each subtree’s root.
    • Nodes with no children are leaves.
    • Nodes with same parent are siblings.
    • If a node v is on the path from r to u, then v is an ancestor of u, and u is a descendant of v.

  • More terminology

    • The depth of a node u is the length of the path from u to r.
    • The height of a node u is the length of the longest path from u to one of its descendants.
      • Height of a leaf node is zero.
      • Height of a non-leaf node is the max height of its children plus one.

Binary Trees

每个节点至多有两个子节点。left child and right child

  • full binary tree

    对于每个节点，要么没有子节点，要么有两个子节点。

    满二叉树要么只有一个根节点，要么根的两个子树都是满二叉树。

  • complete binary tree

    除了最后一层每一层都有最多的节点，且最后一层尽可能的在树的左半边。

    可以用数组表示。

  • Perfect binary tree

    所有的非叶子的节点都有两个子节点，并且所有的叶子节点都有相同的深度。

    ps：CLRS call perfect binary trees as complete binary trees.

Tree Traversal

  • Preorder traversal

    从根部节点开始访问，然后访问子节点下的子树。

  • Postorder traversal

    先访问子节点下的子树，然后访问根部节点。

  • Inorder traversal

    先访问左子节点下的子树，接着访问根部节点，最后访问右边子节点下的子树。

Complexity of recursive traversal

  • Time complexity

    $\Theta(n)$ to process every node.

  • Space complexity

    $O(n)$ as the most depth of call stack is $\Theta(n)$

Search Trees

Set ADT

used to represent a set of elements with (usually distinct) key values.

Each element has a key field and a data field.

  • Operation

    • Search(S, k): Find an element in S with key value k.
    • Insert(S, x): Add x to S. (What if element with same key exists?)
    • Remove(S, x): Remove element x from S, assuming x is in S.
    • Remove(S, k): Remove element with key value k from S.

    如果是有序元素：

    • Min(S)\Max(S): Find minimum/maximum element in S.
    • Successor(S, x) or Successor(S, k): Find smallest element in S that is larger than x.key (or key k).
    • Predecessor(S, x) or Predecessor(S, k): Find largest element in S that is smaller than x.key (or key k).

Binary Search Tree

每个节点存储一个元素且满足BST propertity。

Search in BST

有一个BST根部节点$x$和key $k$，如何通过$k$找到一个元素。

  • 如果$x.key = k$就直接返回$x$
  • 如果$x.key>k$就在$x.left$下的子树下进行递归。
  • 如果$x.key<k$就在$x.right$下的子树下进行递归。

是一个尾递归的过程，所以可以用迭代表示出来。

  • Complexity

    $\Theta(h)$ where ℎ is the height of the BST.

    • How large can h be in an n-node BST?

      $\Theta(n)$ When BST is like a path\line.

    • How small can h be in an n-node BST?

      $\Theta(logn)$ When BST is well-balanced.

Min and Max in BST

  • Find a min

    向左走到底。

  • Find a max

    向右走到底。

  • Time complexity

    $\Theta(h)$ In the worst case.

Successor in BST

  • BSTSuccessor(x)

    Find the smallest element in the BST with key value larger than x.key.

    • 如果右边子树非空，右边子树中最小的元素就是结果。

    • 如果右边子树为空，结果就是离x最近而且左子节点也是x祖先节点的祖先节点。

    • BSTSuccessor(x):
if (x.right!=NULL)
  return BSTMin(x.right)
y = x.parent
while (y!=NULL and y.right==x)
  x = y
  y = y.parent
return y

  • Time complexity

    $\Theta(h)$ in the worst case.

Remove in BST

  • BSTRemove(T, z)

    • Case 1: z没有子节点。

      直接移除z节点。($\Theta(1)$)

    • Case 2: z只有一个子节点。

      Elevate subtree rooted at z’s single child to take z’s position.($O(h)$)

    • Case 3: z有两个子节点。

      • Case a: z的右节点的左节点为空
      • Case b: z的右节点的左节点非空

      Replace node z with min value node in subtree rooted at z.rigℎt.

      Replace node z with BSTSuccessor(z)

      BSTSuccessor(z) can be: r if r.left=NULL; BSTMin(r.left) if r.left≠NULL

Height of BST

The expected height of a randomly built BST is O(logn).

Treap

A Treap is a binary tree in which each node has a key value, and a priority value.

  • The key values must satisfy the BST-property

    左边小于右边。

  • The priority values must satisfy the MinHeap-property

    顶部最小。

  • Insert in Treap

    • 给待添加节点分配随机权重。
    • 根据BST-property插入节点。
    • 维护Min-heap的性质

  • Remove in Treap

    • 将节点下沉直到成为叶子节点。
    • 移除叶子。
