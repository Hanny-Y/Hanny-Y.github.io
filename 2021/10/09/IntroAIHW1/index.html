摘要:

本次作业使用一种推箱子游戏“Bait”，通过深度优先搜索、受限深度优先搜索和最佳优先搜索(A*) 分别自主实现推箱子游戏，并尝试理解通过 MCTS 控制游戏。本文主要介绍了前三种算法的自主实现过程，包括实验思路和具体实现，并尝试对第四种控制方式进行理解和简要介绍。

关键词:

深度受限搜索;受限深度优先搜索;最佳优先搜索;蒙特卡洛树搜索;

1 引言

GVG-AI 框架是为了通用人工智能的研究开发的游戏框架，基于 VGDL（视觉游戏描述语言），能够构成多种游戏。

2 自主实现

2.1 深度优先搜索

2.1.1 算法介绍

DFS 是一种用于遍历或搜索树或图的算法。这个算法会尽可能深的搜索树的分支。当节点 v 的所在边都 己被探寻过，搜索将回溯到发现节点 v 的那条边的起始节点。这一过程一直进行到已发现从源节点可达的所 有节点为止。如果还存在未被发现的节点，则选择其中一个作为源节点并重复以上过程，整个进程反复进行 直到所有节点都被访问为止。

DFS 的实现较为简单，直接进行搜索，将可行的动作都保存到 all_path 中，在 act 函数中，每次返回一个 动作即可。在本次任务中犯的错误是，要输出动作的下标一开始没有初始化，后来修正。

2.2 受限深度优先搜索

2.2.1 算法介绍

DFS 设定一个最大深度 limitDepth，当搜索深度大于 limitDepth 的时候立即回溯，放弃当前搜索路径，转而去探索其他节点，从而避免了在无穷状态空间中陷入深度无限的分支。

在任务 2 中，主功能函数 limitDepthSearch 主要用于返回一个代价值，而最小代价值是通过每一步的代价比较得出的最小值。代价值的计算采用了一个较为简单的启发函数，即为 avatar 到 key 的距离与 key 到 goal 的距离之和。在主功能函数中，所有的子情况都应该有相应的返回值，所以经过考虑，设置了一个很大的数 字 10000 作为任务失败的返回值，将 0 作为任务成功的返回值，如果在一个状态下搜索深度已经达到已定的最大值，此时 esValue 函数计算预计代价值并作为函数结果返回到 act 函数里与当前最小代价值进行比较，如 果比最小代价值小则更新最小代价值，最后 act 得到代价值最小的动作并返回。

最初的最大搜索深度设置为 20，无法通过第一关，所以进行了缩小范围，发现最大可行值为 7，恰好可 以通过第一关，但无法通过后面的关卡。

2.3 最佳优先搜索(A*)

2.3.1 算法介绍

Astar是一种图遍历和路径搜索算法，由于其完备性、最优性和最优效率，它经常被用于计算机科学的许多领域。Astar是一种知情搜索算法，或最佳优先搜索，这意味着它是根据加权图制定的：从图的特定起始节点开始，它旨在找到具有最小的给定目标节点的路径成本（最短行驶距离、最短时间等）。它通过维护源自起始节点的路径树并一次一条边扩展这些路径直到满足其终止标准来实现这一点。在其主循环的每次迭代中，A* 需要确定要扩展的路径。它是根据路径的成本和将路径一直延伸到目标所需的成本估计来实现的。具体来说，A* 选择最小化的路径 f(n)=g(n)+h(n)，其中 n 是路径上的下一个节点，g ( n )是从起始节点到 n 的路径成本，而 h ( n )是一个启发式函数，用于估计从 n 到目标的最便宜路径的成本。当 A* 选择扩展的路径是从起点到目标的路径时，或者没有符合扩展条件的路径时，A* 终止。启发式函数是特定于问题的。如果启发式函数是可接受的，这意味着它永远不会高估达到目标的实际成本，A* 保证返回从开始到目标的最小成本路径。

在任务三中，主要利用了一个优先队列 open_node 来存储相关节点（开放集），对于节点的成本分手中有钥匙和手中没有钥匙两种情况在物体之间曼哈顿距离之和的基础上构造了启发函数，如果已经拿到了钥匙，成本为精灵到目标点的曼哈顿距离加上执行动作的额外成本，如果还没有拿到钥匙，则要在上述基础上再加上精灵到钥匙位置的距离。接下来对 open_node 进行遍历，选取成本最低的 node 加入到过去走过的状态中，如果已经历过该状态，进行一定更新。

2.4 蒙特卡洛树搜索

蒙特卡洛树搜索是一种用于某些决策过程的启发式搜索算法。

一般说来，蒙特卡洛树搜索的每个循环包括四个步骤：

1.选择：从根节点 R 开始，连续向下选择子节点至叶子节点 L。下文将给出一种选择子节点的方法，让游戏树向最优的方向扩展，这是蒙特卡洛树搜索的精要所在。

2.扩展：除非任意一方的输赢使得游戏在 L 结束，否则创建一个或多个子节点并选取其中一个节点 C。

3.仿真：在从节点 C 开始，用随机策略进行游戏。

4.反向传播：使用随机游戏的结果，更新从 C 到 R 的路径上的节点信息。

在本次任务四中，已实现的代码中，首先将最初的状态录入根节点，接着对树中已有的节点进行拓展，从根部开始，如果该节点还有子节点处在未被拓展的状态，则拓展子节点，若已全部拓展，则进入 uctvalue 最大的子节点直到其有子节点未被拓展。对于将要被拓展的节点，在该节点的子节点中进行随机选择，并将此节点加入蒙特卡洛树，加入树以后，若还未达到最大搜索深度则在当前可执行的动作中随机选择并执行直到游戏结束或者达到最大搜索深度并返回 value 值。接着对该节点进行向上回溯，计算 totValue。最后返回的 act应该是一个 mostVisited 的节点且价值最高的操作。
